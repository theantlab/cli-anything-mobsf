# cli-anything-mobsf

CLI harness for [MobSF (Mobile Security Framework)](https://github.com/MobSF/Mobile-Security-Framework-MobSF) — built with the [CLI-Anything](https://github.com/hkuds/cli-anything) methodology.

Wraps the MobSF REST API and integrates local reverse engineering tools into a single command-line workflow for mobile app security analysis.

## Features

- **Full analysis pipeline** — one command to upload, scan, decompile, analyse native libs, map the attack surface, test repackaging, build AppShield protection, and patch with Objection/Frida
- **Intelligent Objection patching** — reads analysis artifacts (APKiD, attack surface, MobSF report) to auto-generate Frida bypass scripts for SSL pinning, root detection, and anti-debug
- **Categorised attack surface scanning** — 22 security categories with risk-ranked findings, file locations, and auto-generated key attack vector summary
- **MobSF API CLI** — upload, scan, report, scorecard, compare, suppress, dynamic analysis, Frida instrumentation
- **REPL mode** — interactive shell with session state, undo/redo
- **JSON output** — `--json` flag on every command for machine-readable output

## Installation

```bash
git clone https://github.com/storriekeir/cli-anything-mobsf.git
cd cli-anything-mobsf
pip install -e .
```

## Configuration

Set your MobSF server URL and API key:

```bash
export MOBSF_URL=http://127.0.0.1:8030
export MOBSF_API_KEY=your_api_key_here
```

Or pass them as flags: `cli-anything-mobsf --url http://... --api-key ...`

The CLI will also attempt to read `~/.MobSF/secret` to derive the key automatically.

## Usage

### Full Analysis Pipeline

```bash
# Analyse an APK — produces a complete analysis directory
cli-anything-mobsf analyse ./app.apk

# Skip stages you don't need
cli-anything-mobsf analyse ./app.apk --skip appshield --skip repackage

# Custom output directory and SDK version
cli-anything-mobsf analyse ./app.apk -o ./results -v 34.0.0
```

The pipeline displays a progress bar with per-stage timing:

```
  Analysing: app.apk
  Output:    ./app_analysis
  Stages:    6/8 (2 skipped)
  ────────────────────────────────────────────────────────

  [██████████░░░░░░░░░░]  50%  3/6  APKiD fingerprinting
    Running local APKiD...
  ✓ apkid                12.4s

  ────────────────────────────────────────────────────────
  Done in 8m 32s  |  6 passed  0 failed  2 skipped
```

Output structure:

```
app_analysis/
├── mobsf/                    # MobSF reports, scorecard, APKiD, PDF
├── decompiled/               # JADX Java source + resources
├── apktool/                  # Smali, class counts, API grep
├── attack_surface/           # Categorised attack surface reports
│   ├── attack_surface_summary.txt
│   ├── attack_surface_report.txt
│   └── attack_surface.json
├── native/                   # readelf, binwalk, strings per ABI
├── repackage/                # Re-sign test with attack keystore
├── appshield/                # APK Defender protected build
├── objection/                # Intelligent Frida-patched APK
│   ├── patching_decisions.json
│   ├── libfrida-gadget.script.so  # Auto-generated bypass script
│   └── *.objection.apk
└── summary.json              # Combined findings + metadata
```

### Standalone Attack Surface Scan

```bash
# Run on an apktool-decoded directory
cli-anything-mobsf attack-surface ./app_apktool/app/
```

### MobSF API Commands

```bash
# Upload and scan
cli-anything-mobsf upload ./app.apk
cli-anything-mobsf scan

# Reports
cli-anything-mobsf report --hash <md5>
cli-anything-mobsf scorecard --hash <md5>
cli-anything-mobsf pdf --hash <md5> -o report.pdf

# Compare two scans
cli-anything-mobsf compare <hash1> <hash2>

# Dynamic analysis
cli-anything-mobsf dynamic start --hash <md5>
cli-anything-mobsf dynamic stop --hash <md5>

# Frida
cli-anything-mobsf frida scripts
cli-anything-mobsf frida instrument --hash <md5> --hooks "ssl_bypass"
```

### REPL Mode

```bash
cli-anything-mobsf
mobsf> upload ./app.apk
mobsf> scan
mobsf> report
mobsf> quit
```

## Attack Surface Categories

The attack surface scanner covers 22 security categories:

| Risk | Categories |
|------|-----------|
| CRITICAL | Network & TLS Configuration, Hardcoded Credentials & Secrets, Financial & Payment |
| HIGH | Cryptography, Authentication & Session, WebView, Insecure Data Storage, IPC & Intent, Sensitive Data & PII, App Integrity & Tamper Detection, Cloud & Backend Services |
| MEDIUM | Root Detection, Emulator & VM Detection, Anti-Debug & Dynamic Analysis, Hooking Framework Detection, Information Leakage via Logging, Clipboard & Screenshot |
| INFO | HTTP Client Libraries, Protection Vendors, Analytics & Third-Party SDKs |

## Intelligent Objection Patching

The objection patcher reads analysis artifacts to make informed decisions:

| Artifact | Informs |
|----------|---------|
| MobSF report | Target class, SDK level, APK size |
| APKiD | Packer detection → ignore nativelibs flag |
| Attack surface | TLS pinning → network security config + SSL bypass script |
| Attack surface | Root detection → RootBeer or generic bypass script |
| Attack surface | Anti-debug → debugger detection bypass script |
| Native libs | Architecture selection from actual ABIs present |
| AndroidManifest | Application class as injection point |

## Dependencies

### Required
- Python >= 3.9
- [MobSF](https://github.com/MobSF/Mobile-Security-Framework-MobSF) running instance
- click, requests (installed automatically)

### For full analysis pipeline
- [jadx](https://github.com/skylot/jadx) — DEX decompiler
- [apktool](https://apktool.org/) — APK disassembler
- [apkid](https://github.com/rednaga/APKiD) — APK fingerprinting
- [binwalk](https://github.com/ReFirmLabs/binwalk) — binary analysis
- [objection](https://github.com/sensepost/objection) — Frida gadget patcher
- [frida](https://frida.re/) — dynamic instrumentation
- readelf, strings — standard binutils

### Optional
- apkdefender (Verimatrix AppShield) — for the appshield stage
